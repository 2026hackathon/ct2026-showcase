## 解决什么问题

Claude Code / Codex 这类个人 code agent 只能帮你「一个人」把代码写得更快，无法替你把从需求到产品、到 QA 验收的「整个活」干完。

## 怎么解决

你以「客户」身份把一句需求告诉前台需求顾问（AE），公司里 CEO、产品、项目经理、技术负责人、工程师、设计师、测试、AE 共 **8 个岗位 Agent** 像真实公司一样自主讨论、开会、产生分歧、拍板决策、动手产出，最终交付从 PRD、技术方案到**真实可运行代码仓库**的完整成果。

## 关键亮点

- **人是客户/观察者，不是操作员**：交一个目标即可，并能在地图、动态流、共享白板上实时观测组织怎么协作。
- **没有写死的剧本**：何时开会、找谁聊、何时产出都由 Agent 每一拍自主判断涌现；客户随时插话会在组织内自然传播。
- **会交付的公司**：QA Agent 对照需求逐项验收，不达标回流重做，而非聊得热闹的群聊。
